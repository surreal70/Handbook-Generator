## Purpose

This document describes the continuous improvement process for IT governance.

## Scope

This document applies to:
- {{ meta-organisation.name }}
- All IT governance processes

## Improvement Cycle

### 1. Plan

- Set improvement goals
- Plan measures
- Allocate resources
- Create timeline

### 2. Do

- Implement measures
- Conduct pilot projects
- Conduct training
- Create documentation

### 3. Check

- Measure results
- Check goal achievement
- Collect feedback
- Lessons learned

### 4. Act

- Standardize successful measures
- Make adjustments
- Share best practices
- Plan next cycle

## Improvement Areas

| Area | Goal | Measures |
