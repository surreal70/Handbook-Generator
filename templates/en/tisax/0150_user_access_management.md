## Purpose

This document describes the processes and procedures for managing user access according to TISAX requirements.

## Scope

This document applies to all user accounts and access at [TODO].

## User Lifecycle

### 1. Onboarding
- Request by HR department
- Approval by supervisor
- Creation of user accounts
- Assignment of standard permissions
- Training before access grant

### 2. Changes
- Review of existing permissions
- Adjustment to new role
- Removal of no longer needed rights
- Documentation of changes

### 3. Offboarding
- Deactivation on last working day
- Return of access credentials
- Archiving of data
- Process documentation

## Permission Management

### Permission Model
- Role-based
- Group-based

### Permission Request Process
1. User or supervisor submits request
2. Business justification required
3. Approval by supervisor
4. Approval by information owner (for data access)
5. Implementation by IT
6. Confirmation to requester

## Access Reviews

### Regular Review
- Quarterly: Review by supervisors
- Annually: Full recertification

### Process
1. Automatic generation of review lists
2. Distribution to supervisors
3. Review and confirmation
4. Escalation if no response
5. Implementation of changes
6. Documentation and reporting

## Privileged Access

### Management
- Separate privileged accounts
- Just-in-time access
- Session recording
- Automatic password rotation

### Monitoring
- Real-time monitoring
- Alerting on anomalies
- Regular auditing
- Compliance reporting

## External Users

### Requirements

### Management
- Request with technical justification
- Approval by IT manager
- Documentation of purpose
- Assignment of responsible person

### Security

- Regular password rotation
- Minimum permissions
- No interactive login
