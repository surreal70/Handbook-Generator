## Scope

This document applies to all employees, contractors, change processes, and mobile devices.

## Human Resources Security (HRS)

### Background Screening

**Screening Types**:
- Identity verification
- Reference checks
- Education verification
- Criminal background check (where legally permitted)

**Screening Scope**:
- All employees
- Contractors with access to sensitive data
- Privileged users

**Screening Frequency**:
- Upon hiring
- Upon role change
- Periodic repetition (every 3-5 years)

### Employment Contracts and Agreements

**Contractual Requirements**:
- Non-Disclosure Agreements (NDA)
- Acceptable Use Policy (AUP)
- Code of Conduct
- Security policies

**Contractor Agreements**:
- Service Level Agreements
- Security requirements
- Data protection clauses
- Liability provisions

### Termination and Role Changes

**Offboarding Process**:
1. Notification to IT and security
2. Access revocation
3. Return of devices and access cards
4. Exit interview
5. Documentation

**Role Changes**:
- Review of access rights
- Adjustment of permissions
- Training for new role
- Documentation

## Security Awareness and Training

### Training Program

**Mandatory Training**:
- Security Awareness Training (annual)
- Data Protection Training (annual)
- Phishing Awareness Training (quarterly)
- Role-specific Training (as needed)

**Training Content**:
- Information security fundamentals
- Data protection and GDPR
- Phishing and social engineering
- Password security
- Mobile device security
- Cloud security best practices
- Incident reporting

### Training Methods

**Training Formats**:
- E-learning
- In-person training
- Webinars
- Phishing simulations
- Security awareness campaigns

**Training Tracking**:
- Attendance registration
- Completion tests
- Certificates
- Compliance reporting

### Phishing Simulations

**Simulation Program**:
- Monthly phishing tests
- Various difficulty levels
- Realistic scenarios

**Metrics**:
- Click rate
- Reporting rate
- Improvement over time

## Change Control and Configuration Management (CCC)

### Change Management Process

**Change Types**:
- Standard Changes (pre-approved)
- Normal Changes (approval required)
- Emergency Changes (expedited procedure)

**Change Process**:
1. Change request
2. Impact assessment
3. Risk assessment
4. Change approval
5. Change implementation
6. Change verification
7. Change documentation

### Change Advisory Board (CAB)

**CAB Members**:
- Change Manager
- IT Operations
- Security Team
- Application Owners
- Business Representatives

**CAB Meetings**:
- Weekly meetings
- Emergency CAB as needed
- Review of all Normal and Emergency Changes

### Configuration Management

**Configuration Management Database (CMDB)**:
- Configuration Items (CIs)
- CI relationships
- CI attributes
- Change history

**Configuration Baselines**:
- Approved configurations
- Security baselines
- Compliance baselines

**Configuration Audits**:
- Regular audits
- Drift detection
- Remediation
